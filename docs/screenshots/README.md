# Screenshot capture guide

The project report (`../PROJECT_REPORT.md`) references the files below. Capture them on the running dashboard and drop them in this folder — the report will render them automatically.

| Filename                | Capture                                                                                                                |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `01-title-bar.png`      | Top sticky header — both rows visible (title + filters; date range below). Zone filters collapsed.                     |
| `02-zone-filters.png`   | Same header but click **▸ ZONE FILTERS** so all four cards (PSM cyan, Sigma purple, Silo yellow, Packaging orange) show. |
| `03-unified-table.png`  | Mid-page table view sorted by `ts ↓`. Make sure at least one CRITICAL pink row, one WARNING yellow row, and one healthy green row are visible. |
| `04-tag-drawer.png`     | Click a PSM RM row (e.g. AOS, Caustic) to open the right-side drawer. Capture with the 10-reading sparkline visible.    |
| `05-theme.png`          | Toggle theme via the navbar, take captures of both dark and light. A simple horizontal collage is fine.                 |

## Recommended capture settings

- Browser zoom: 100 %
- Window width: 1920 px (or larger). The title bar layout is optimised for ≥ 1600 px wide.
- Use the browser's full-page screenshot mode for `01` and `02` so the entire sticky header is in one frame.
- Crop tightly — there's no need to show the OS chrome.

## Tools that work well on Windows

- Built-in: `Win + Shift + S` (Snipping Tool — rectangle mode).
- Browser DevTools: Chrome `Ctrl + Shift + P` → "Capture full-size screenshot" for full-page shots.
